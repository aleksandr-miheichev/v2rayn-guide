## Общие вопросы

??? question "Какую версию v2rayN использовать?"

    Рекомендуется **последняя стабильная версия** с
    [github.com/2dust/v2rayN/releases](https://github.com/2dust/v2rayN/releases).

    Убедитесь, что у вас версия с поддержкой sing-box 1.12+
    (если планируете использовать TUN-режим).

??? question "Xray или sing-box — что выбрать?"

    - **Xray** — стандартное ядро, работает в режиме системного прокси.
      Подходит для большинства задач.
    - **sing-box** — используется в **TUN-режиме**, когда нужно перехватывать
      весь трафик системы (включая приложения, которые не поддерживают прокси).

    Рекомендуется настроить DNS для обоих — переключение между ядрами не
    потребует дополнительной настройки.

??? question "Что такое TUN-режим?"

    TUN создаёт **виртуальный сетевой интерфейс** на уровне ОС.
    Весь трафик системы проходит через него — даже от приложений,
    которые не умеют работать с прокси (игры, некоторые мессенджеры).

    Минус: требует прав администратора и немного больше ресурсов.

## Проблемы с маршрутизацией

??? question "Сайт открывается, но думает что я за рубежом"

    Сайт идёт через прокси вместо напрямую. Решение:

    Добавьте домен сайта в правило **«Российские сайты напрямую»**
    (правило 13 в нашей настройке) — оно использует `geosite:category-ru`.
    Если домена нет в этом списке, добавьте его вручную отдельным правилом
    с `outboundTag: direct`.

??? question "Заблокированный сайт не открывается"

    1. Проверьте, что прокси-сервер работает (иконка в трее v2rayN зелёная)
    2. Проверьте, что сайт есть в `geosite:ru-blocked`
    3. Если нет — добавьте домен вручную в правило 11 (заблокированные домены)
    4. Убедитесь, что финальное правило (`0-65535 → proxy`) на месте

??? question "Всё идёт через прокси, ничего напрямую"

    Проверьте **порядок правил**. Правила `direct` (приватные сети,
    российские сайты) должны стоять **выше** правил `proxy` и финального
    правила.

    Также убедитесь, что файлы `geosite.dat` / `geoip.dat` актуальны
    и содержат нужные списки.

??? question "Авито / банк / госуслуги не работают"

    Эти сайты блокируют зарубежные IP. Они уже должны быть в списке
    `geosite:category-ru` (правило 13 — российские сайты напрямую).
    Если конкретный сайт всё равно не работает — добавьте его домен
    вручную в отдельное правило с `outboundTag: direct`.

## Проблемы с DNS

??? question "DNS Leak Test показывает серверы провайдера"

    У вас DNS-утечка. Проверьте по порядку:

    1. **Правило блокировки QUIC (UDP:443)** — должно быть **первым** в списке
       маршрутизации
    2. **Настройки DNS** — должны быть заполнены (Quad9 + Mullvad)
    3. **Перезапуск** — полностью перезапустите v2rayN (не только переподключение)
    4. **Системный DNS** — в настройках Windows проверьте, что DNS не
       переопределён вручную

??? question "Сайты грузятся медленно после настройки DNS"

    DoH через прокси добавляет задержку на первый запрос к домену.
    После кэширования (8192 записи в нашей конфигурации) — повторные
    запросы будут быстрыми.

    Если совсем медленно — проверьте latency до вашего прокси-сервера:
    ```
    ping ваш-сервер.example.com
    ```

    Нормально: < 100 мс. Если > 200 мс — рассмотрите сервер поближе.

??? question "Что такое DNS-петля и как её избежать?"

    DNS-петля: v2rayN пытается зарезолвить `dns.quad9.net` через
    `dns.quad9.net` → бесконечный цикл → ничего не работает.

    Наша конфигурация **уже решает** эту проблему:

    - **Xray:** блок `hosts` привязывает `dns.quad9.net` к IP `9.9.9.9`
    - **sing-box:** подключение напрямую по IP `9.9.9.9` с `Host` заголовком

## Telegram

??? question "Telegram работает, но звонки не проходят"

    Telegram использует UDP для звонков. Проверьте:

    1. Правило «Telegram UDP» есть в маршрутизации
    2. Сеть (Network) установлена в `udp`
    3. IP-диапазоны Telegram актуальны
       ([проверить](https://core.telegram.org/resources/cidr.txt))

??? question "Telegram не подключается вообще"

    1. Проверьте, что прокси работает (откройте youtube.com)
    2. В Telegram: Настройки → Данные и память → Тип соединения →
       должно быть «Использовать системный прокси» или настроен SOCKS5

## Прочее

??? question "Торренты медленные"

    По нашей конфигурации торренты идут **напрямую** (правило BitTorrent).
    Но для этого нужен включённый **sniffing** (определение протоколов)
    в v2rayN:

    **Настройка → Основные настройки → Sniffing** → включить

    Без sniffing v2rayN не распознаёт BitTorrent и отправляет трафик
    через прокси (финальное правило).

??? question "Как обновить списки geosite/geoip?"

    - **Xray:** v2rayN обычно обновляет их автоматически, или:
      v2rayN → Главное меню → Обновить → Обновить Geo-файлы
    - **sing-box:** rule_set обновляются автоматически каждые 6 часов
      (параметр `update_interval` в конфигурации)

??? question "Можно ли использовать другие DNS вместо Quad9/Mullvad?"

    Да. Требования к DNS-серверу:

    - Поддержка DoH (DNS over HTTPS)
    - Политика no-log (или приемлемая для вас политика)
    - Стабильная работа

    Альтернативы: AdGuard DNS (`dns.adguard-dns.com`), NextDNS.

    Но не используйте Google DNS или провайдерский DNS — теряется весь
    смысл настройки.
