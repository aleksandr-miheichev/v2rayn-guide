# Как устроены правила маршрутизации

Прежде чем настраивать — разберёмся с терминами. Не пугайтесь, всё проще
чем кажется.

---

## Три направления трафика

Каждое правило отправляет трафик в одно из трёх «направлений» (`outboundTag`):

| outboundTag | Что делает | Пример |
|---|---|---|
| `proxy` | Отправляет через ваш прокси-сервер | YouTube, Google, Instagram |
| `direct` | Отправляет напрямую (без прокси) | VK, Яндекс, локальная сеть |
| `block` | Полностью блокирует соединение | Реклама, телеметрия |

---

## Как v2rayN определяет куда отправить

Правило может проверять:

### :material-web: Домен (`domain`)

Проверяет **имя сайта** в адресной строке.

```
domain: ["youtube.com", "google.com"]  →  outboundTag: proxy
```

Означает: если сайт — `youtube.com` или `google.com` — отправить через прокси.

### :material-ip-network: IP-адрес (`ip`)

Проверяет **IP-адрес** сервера, к которому идёт подключение.

```
ip: ["91.108.4.0/22"]  →  outboundTag: proxy
```

Означает: если IP попадает в диапазон `91.108.4.0/22` (это Telegram) — через прокси.

### :material-network: Порт и протокол (`port`, `network`)

```
port: "53", network: "udp"  →  outboundTag: block
```

Означает: заблокировать все DNS-запросы по UDP на порт 53 наружу
(чтобы предотвратить DNS-утечки).

## Что такое `geosite` и `geoip`

Это **готовые списки** доменов и IP-адресов, которые поддерживает сообщество:

<div class="param-card" markdown>

**`geosite:ru-blocked`** — список доменов, заблокированных Роскомнадзором
(YouTube, Instagram, Twitter и тысячи других).
Обновляется автоматически.

</div>

<div class="param-card" markdown>

**`geosite:ru-available-only-inside`** — сайты, которые работают **только из России**
(некоторые госсервисы, региональные ресурсы).
Их нужно пускать напрямую, иначе они не откроются через зарубежный прокси.

</div>

<div class="param-card" markdown>

**`geosite:category-ads-all`** — домены рекламных сетей и трекеров.
Блокировка = чище и быстрее интернет.

</div>

<div class="param-card" markdown>

**`geoip:private`** — приватные IP-диапазоны (`192.168.x.x`, `10.x.x.x`, `127.x.x.x`).
Это ваша локальная сеть — всегда напрямую.

</div>

<div class="param-card" markdown>

**`geoip:ru-blocked`** — IP-адреса заблокированных ресурсов.
Дополняет `geosite:ru-blocked` — ловит трафик даже когда домен не резолвится.

</div>

!!! info "Откуда берутся эти списки?"

    Мы используем списки из проекта
    [runetfreedom/russia-v2ray-rules-dat](https://github.com/runetfreedom/russia-v2ray-rules-dat) —
    они поддерживаются сообществом и регулярно обновляются.

## Порядок проверки правил

```
Запрос → Правило 1 (совпало?)
              ↓ нет
         Правило 2 (совпало?)
              ↓ нет
         Правило 3 (совпало?)
              ↓ нет
           ...
         Финальное правило → proxy
```

!!! tip "Принцип"

    - **Более узкие** правила — выше (конкретные порты, конкретные домены)
    - **Более широкие** — ниже (целые категории `geosite`)
    - **Финальное** правило — всегда последнее (ловит всё, что не попало ни в одно правило)
